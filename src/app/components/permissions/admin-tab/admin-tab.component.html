Admin Setup

Guidance + deep links for configuring Keycloak so role grants flow through groups, and so onboarding can be delegated safely.

open_in_new Keycloak Admin Console
tips_and_updates Recommended Keycloak configuration How to set up auto-role-assignment via groups, with delegated onboarding
  1. Define realm roles for each broad permission tier your project needs (e.g. polari-admin, polari-developer, polari-user, polari-viewer). These come pre-seeded; new roles you add in KC sync into Polari's Roles tab within ~60s.
  2. Create a group per onboarding intent (e.g. Polari Administrators, Polari Developers, Polari Users). On each group, attach the realm roles that group should grant via the group's Role Mappings tab. New members inherit those roles in their JWT immediately.
  3. Turn on fine-grained admin permissions on the realm (Realm Settings → General → Admin permissions). This creates an admin-permissions client whose authorization tab is where per-resource policies live.
  4. Create an access-management role (e.g. polari-onboarder) and grant it only the "manage members" permission on the relevant groups via policies — not role editing, not realm settings. Intermediate admins holding this role can add users to groups but can't touch the code-modifying roles.
  5. Configure grants on the Roles tab in Polari. For each role, set which CRUDE ops it grants per object class. Polari's backend reads these and enforces them on every request.
